Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 13 and Pull Request 97. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

The user can select from a choice of valid input values, unless there are more than forty valid values for any distinct piece of data within the input field.

Alternative formats of separator characters such as dash, dots, bracket and spaces are accepted in numerical inputs.
