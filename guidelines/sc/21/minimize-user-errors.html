Minimize user errors

A

Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 13 and Pull Request 97. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

Where it is possible to provide pre-defined choices, the user can select from a list of valid input values. Where users must manually enter numerical inputs, alternative separator characters are accepted.

Exceptions:
  • Where pre-defined choices for a piece of data have more than thirty-one valid values, it is not necessary to provide pre-defined choices. For example, a piece of data with more then 31 choices could be a year in a date field.
  • Where pre-defined choices interfere with the main purpose of the content.
  • Where alternative separator characters could alter the numeric value, such as substituting commas with periods.
