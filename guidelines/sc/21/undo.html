Undo

A

Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 38. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

Users are provided with the ability to undo an action and to correct mistakes such that:

  • The user can return to the location they were at, prior to the change in context, via labeled controls without unwanted loss of data unless the data loss is part of the correction; or
  • the user can repair information entered, via labeled controls, and without unwanted loss of data unless the data loss is part of the correction.
Exceptions:
  • Where allowing the user to undo an action or maintaining data may cause harm such as adding risk to the user privacy or security.
  • Where the user has confirmed an action it does not have to be reversible.
  • Where allowing the user to undo an action may interfere with the essential function of the content.
  • Where the action can no longer be controlled by the site.
