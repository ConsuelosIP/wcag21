Undo

A

Proposed

This is a proposed Success Criterion that has not yet been formally approved by the Working Group. Discussion of the issue is available in Issue 38. To file comments on this proposal, please raise new issues for each discrete comment in GitHub.

Users can undo actions, return to the previous context and correct data entry without loss of non-dependent data except when:

  • allowing the user to undo an action or maintaining data may cause harm such as adding risk to the user privacy or security;
  • the user has confirmed an action;
  • allowing the user to undo an action may invalidate the activity;
  • the action can no longer be controlled by the site;
  • the user has been idle for 24 hours.
